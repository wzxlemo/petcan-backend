# Pet App - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
宠物能APP

### 1.2 产品定位
宠物智能小管家

### 1.3 核心价值

🌟 **情感联结 + 功能黏性**
- 为宠物创建专属虚拟形象
- 让宠物通过语音"回应"主人
- 智能管理宠物健康与日程
- 硬件数据驱动的情感化交互

🌟 **ToC米家硬件 + ToB宠物专属小龙虾🦞**
- 面向C端：打造宠物智能设备生态
- 面向B端：为宠物品牌/机构提供定制化解决方案

🌟 **未来想象空间**
- **内容入口**：与腾讯宠物内容生态联动
- **社交延伸**：从宠物社交到人的社交（"叫妈妈"建立情感纽带）

---

## 2. 用户流程

### 2.1 核心流程（MVP）

```
微信登录 → 添加宠物信息 → 上传照片 → AI生成2D形象 → 
语音录入训练 → 选择应答词 → 确认语音 → 进入主页
```

### 2.2 流程时长目标
- 新用户首次完成流程 ≤ 1分钟

### 3.2.2 照片上传（真实相机/相册调用）

| 功能 | 需求 | 优先级 |
|------|------|--------|
| 全身照上传 | 调用设备相机或相册选择，支持预览 | P0 |
| 正脸照上传 | 调用设备相机或相册选择，支持预览 | P0 |
| 图片预览 | 上传后显示预览，支持删除重传 | P0 |
| 文件校验 | 仅支持图片格式，最大5MB | P0 |
| 上传交互 | 点击上传区域触发系统文件选择器 | P0 |

**上传流程：**
```
用户点击上传区域 → 唤起系统相机/相册 → 选择图片 → 本地预览显示 → 校验通过 → 下一步按钮启用
```

**技术实现：**
- 使用 `<input type="file" accept="image/*">` 调用系统文件选择器
- 在移动设备上会自动唤起相机/相册选项
- 使用 FileReader API 读取图片并显示本地预览
- 上传的图片暂存于内存，生成形象后释放

### 3.3 AI 2D形象生成（Lovart API）

| 功能 | 需求 | 优先级 |

**AI生成流程：**
```
用户点击"开始生成" → 调用 Lovart /v1/design/generate → 获取 designId → 轮询 /v1/design/{id} → 生成完成 → 展示结果
```

**Lovart API 配置：**

- **模型**: nano-banana-pro
- **风格**: kawaii卡通风格

**生成状态管理：**
| 状态 | UI展示 | 用户操作 |
|------|--------|----------|

| 失败 | 错误图标 + 错误原因 + 重试/重新上传按钮 | 重试或重新上传 |

**AI接入约束：**
- [x] 已接入 Lovart API，使用 nano-banana-pro 模型
- [x] 失败兜底提示（错误状态UI）
- [x] 仅支持1种风格（MVP）：kawaii卡通
- [ ] 每用户每日生成次数限制（待确定具体次数）

### 3.5 语音合成（TTS）

#### 3.5.1 音色参数计算引擎（Pet Voice Engine）

**核心逻辑**：根据宠物档案（品种、年龄、性别、绝育）计算专属音色参数

**计算公式：**
```
总音调 = 基准音调(100) + 品种基线 + 年龄因子 + 性别因子 + 绝育因子
```

**各因子映射表：**

| 维度 | 条件 | Pitch 变化 | 说明 |
|------|------|-----------|------|
| **品种基线** | 小型犬（泰迪/吉娃娃） | +15~+20 | 体型越小音调越高 |
| | 中型犬（金毛/拉布拉多） | 0 | 基准 |
| | 大型犬（德牧/罗威纳） | -10~-12 | 体型越大音调越低 |
| | 猫类 | +5~+15 | 猫音调普遍偏高 |
| **年龄** | 幼年(0-6月) | +25 | 幼音高亢 |
| | 少年(6-12月) | +15 | |
| | 青年(1-2岁) | +5 | |
| | 成年(2-7岁) | 0 | 基准 |
| | 中年(7-10岁) | -5 | |
| | 老年(10岁+) | -15 | 老音低沉 |
| **性别** | 雌性 | +5 | 雌性偏高 |
| | 雄性 | 0 | 基准 |
| **绝育** | 雄性已绝育 | +10 | 雄性激素下降，声音变柔和 |
| | 雌性已绝育 | +3 | 变化较小 |
| | 未绝育 | 0 | 基准 |

**示例计算：**
- 咪咪（布偶猫，雌性，1岁，未绝育）→ Pitch = 100+12+5+5+0 = **122**
- 旺财（金毛，雄性，3岁，已绝育）→ Pitch = 100+0+0+0+10 = **110**
- 大壮（德牧，雄性，8岁，未绝育）→ Pitch = 100-10-5+0+0 = **85**

#### 3.5.2 TTS Prompt 生成

**Prompt 模板：**
```
一只[年龄描述][品种]宠物，[音色特征词]的声音，用[语气词]喊"[称呼]"
```

**示例：**
- 幼年雌性未绝育泰迪喊"妈妈"：
  `一只年轻的泰迪狗狗，清脆、明亮、稚嫩、活泼、温柔的声音，用亲切可爱的语气喊"妈妈"`

#### 3.5.3 未知生日处理

- 用户选择"未知"后，若选择了年月，显示为"2024年3月"而非"未知"
- 后台抓取年月计算年龄，用于音色参数
- 若完全未选，使用默认成年参数

### 3.6 AI Agent（小能）

**产品定位**：不是"和宠物对话"，而是"宠物专属健康管家"

**类比理解：**
- ❌ 不是：让宠物"说话"的玩具（市面上已有）
- ✅ 而是：24小时在线的"宠物医生+管家"AI助手

#### 3.6.1 功能定义

| 能力 | 说明 | 示例 |
|------|------|------|
| **健康记录** | 通过对话记录饮食/排便/异常 | "旺财今天拉稀了"→记录腹泻事件 |
| **疫苗管理** | 根据品种标准周期提醒 | "上次疫苗8个月前"→计算下次提醒时间 |
| **品种知识** | 不同品种的健康建议 | "金毛容易髋关节问题"→定期关注 |
| **主动询问** | 对未记录信息主动追问 | "今天旺财吃饭正常吗？" |

#### 3.6.2 技术实现方案：OpenClaw + Kimi K2.5（已购买 Allegretto）

**用户现状：** 已购买 Kimi Allegretto 订阅（$39/月），优先使用 Kimi API，不额外引入 GPT。

**推荐方案：OpenClaw 框架 + Kimi K2.5 模型**

| 维度 | OpenClaw + Kimi K2.5 | KimiClaw 托管 | 直接 Kimi API |
|------|---------------------|--------------|-------------|
| **模型** | ✅ Kimi K2.5（已订阅） | ✅ Kimi K2.5 | ✅ Kimi K2.5 |
| **Function Call** | ✅ 完整支持 | ✅ 支持 | ✅ 支持 |
| **数据隐私** | ✅ 自托管，数据留自己服务器 | ❌ Moonshot 云端 | ⚠️ 在 Moonshot |
| **成本** | ✅ 已包含在 Allegretto 订阅 | ❌ 需额外订阅 KimiClaw | ✅ API 按量 |
| **微信接入** | ✅ 容易 | ⚠️ 需额外配置 | ✅ 容易 |
| **定制能力** | ✅ 完全控制 System Prompt | ⚠️ 受限于平台 | ✅ 完全控制 |

**结论：使用 OpenClaw 框架 + Kimi K2.5 API，自托管在我们的 Render 后端上**

**Kimi K2.5 优势（适合"小能"）：**
- 256K 上下文窗口（处理长对话历史）
- 原生多模态（后续可发宠物照片让AI分析）
- 中文理解强（宠物主人多用中文交流）
- 已订阅，无额外成本

#### 3.6.3 "小能" System Prompt 设计

```
你是"小能"，一位专业的宠物健康管家助手。你的职责是帮助宠物主人科学养宠。

【核心能力】
1. 健康记录：当主人描述宠物状况时（如饮食、排便、精神状态），结构化记录到数据库
2. 疫苗管理：根据品种标准周期，提醒疫苗/驱虫/体检时间
3. 品种知识：提供针对性的养护建议
4. 异常预警：识别异常状况并建议就医

【品种知识库示例】
- 金毛：易患髋关节发育不良，建议控制体重，避免剧烈运动
- 布偶猫：心脏肥厚风险，建议定期心超
- 泰迪：髌骨脱位高发，避免频繁站立

【对话风格】
- 温暖亲切，像一位有经验的宠物医生朋友
- 用 emoji 增加亲和力
- 主动关心，不只被动回答

【数据记录格式】
当主人提到具体事件时，调用 record_event 工具：
{
  "type": "diet|exercise|poop|pee|symptom|vaccine|medication",
  "pet_id": "当前宠物ID",
  "description": "事件描述",
  "severity": "normal|mild|moderate|severe",
  "timestamp": "ISO时间"
}
```

#### 3.6.4 疫苗提醒逻辑

```
用户：上次打疫苗是什么时候？
主人：8个月前

小能处理：
1. 查询品种标准周期（金毛核心疫苗每年1次）
2. 计算：上次8个月前 → 已过8个月 → 还剩4个月
3. 存入 vaccine_records 表
4. 设置提醒：4个月后推送
5. 回复："收到！金毛每年需接种核心疫苗。距离上次已8个月，建议4个月后接种。已为你设置提醒。"
```
