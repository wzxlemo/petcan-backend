# Pet App - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
宠物能APP

### 1.2 产品定位
宠物智能小管家

### 1.3 核心价值

🌟 **情感联结 + 功能黏性**
- 为宠物创建专属虚拟形象
- 让宠物通过语音"回应"主人
- 智能管理宠物健康与日程
- 硬件数据驱动的情感化交互

🌟 **ToC米家硬件 + ToB宠物专属小龙虾🦞**
- 面向C端：打造宠物智能设备生态
- 面向B端：为宠物品牌/机构提供定制化解决方案

🌟 **未来想象空间**
- **内容入口**：与腾讯宠物内容生态联动
- **社交延伸**：从宠物社交到人的社交（"叫妈妈"建立情感纽带）

---

## 2. 用户流程

### 2.1 核心流程（MVP）

```
微信登录 → 添加宠物信息 → 上传照片 → AI生成2D形象 → 
语音录入训练 → 选择应答词 → 确认语音 → 进入主页
```

### 2.2 流程时长目标
- 新用户首次完成流程 ≤ 1分钟

## 3. 功能需求

### 3.1 登录模块

| 功能 | 需求 | 优先级 |
|------|------|--------|
| 微信登录 | 跳转微信授权，获取OpenID | P0 |
| 手机号绑定 | 强制绑定，用于找回账号和短信提醒 | P0 |
| 未登录限制 | 未登录不可使用任何功能 | P0 |

### 3.2 宠物管理

| 功能 | 需求 | 优先级 |
|------|------|--------|
| 添加宠物 | 类型、品种、昵称、生日、绝育状态 | P0 |
| 照片上传 | 全身照、正脸照 | P0 |
| 多宠物支持 | 上限20只，下拉切换 | P0 |
| 修改信息 | 支持编辑宠物资料 | P1 |

#### 宠物信息字段
- 类型：猫 / 狗狗
- 品种：下拉选择（猫19种 / 狗狗25种）
- 昵称：最多10字符，不支持符号
- 性别：弟弟 / 妹妹
- 生日：年-月-日（日可选"未知"）
- 绝育状态：未绝育 / 已绝育 / 未知

### 3.2.2 照片上传（真实相机/相册调用）

| 功能 | 需求 | 优先级 |
|------|------|--------|
| 全身照上传 | 调用设备相机或相册选择，支持预览 | P0 |
| 正脸照上传 | 调用设备相机或相册选择，支持预览 | P0 |
| 图片预览 | 上传后显示预览，支持删除重传 | P0 |
| 文件校验 | 仅支持图片格式，最大5MB | P0 |
| 上传交互 | 点击上传区域触发系统文件选择器 | P0 |

**上传流程：**
```
用户点击上传区域 → 唤起系统相机/相册 → 选择图片 → 本地预览显示 → 校验通过 → 下一步按钮启用
```

**技术实现：**
- 使用 `<input type="file" accept="image/*">` 调用系统文件选择器
- 在移动设备上会自动唤起相机/相册选项
- 使用 FileReader API 读取图片并显示本地预览
- 上传的图片暂存于内存，生成形象后释放

### 3.3 AI 2D形象生成（Lovart API）

| 功能 | 需求 | 优先级 |
|------|------|--------|
| 照片生成 | 调用 Lovart API 生成2D风格形象 | P0 |
| 重新生成 | 支持重新上传照片生成 | P0 |
| 风格限制 | MVP仅支持1种风格（kawaii卡通） | P0 |
| 失败处理 | 明确错误提示，支持重试 | P0 |
| 生成进度 | 实时显示生成进度和状态文案 | P0 |
| 结果展示 | 生成成功后展示2D形象图片 | P0 |

**AI生成流程：**
```
用户点击"开始生成" → 调用 Lovart /v1/design/generate → 获取 designId → 轮询 /v1/design/{id} → 生成完成 → 展示结果
```

**Lovart API 配置：**
- **服务商**: Lovart (https://lovart.pro)
- **模型**: nano-banana-pro
- **风格**: kawaii卡通风格
- **Prompt模板**: 根据宠物类型（猫/狗）和品种动态构建

**生成状态管理：**
| 状态 | UI展示 | 用户操作 |
|------|--------|----------|
| 生成中 | 进度条 + 状态文案（连接中/生成中/绘制中） | 可跳过 |
| 成功 | 展示生成的2D形象图片 + 下一步按钮 | 进入语音录制 |
| 失败 | 错误图标 + 错误原因 + 重试/重新上传按钮 | 重试或重新上传 |

**AI接入约束：**
- [x] 已接入 Lovart API，使用 nano-banana-pro 模型
- [x] 失败兜底提示（错误状态UI）
- [x] 仅支持1种风格（MVP）：kawaii卡通
- [ ] 每用户每日生成次数限制（待确定具体次数）

### 3.4 语音系统

| 功能 | 需求 | 优先级 |
|------|------|--------|
| 语音录入 | 系统原生录音，录制两次 | P0 |
| 应答词选择 | 妈妈/爸爸/姐姐/哥哥/主人 | P0 |
| 语音合成 | 讯飞TTS，宠物专属音色 | P0 |
| 宠物铃 | 长按发语音，宠物语音回应 | P0 |

**音色要求：**
- 基于用户录入语音克隆专属音色
- 音色保持一致，音调可根据活跃度变化
- 回应内容AI根据上下文生成

### 3.5 主页功能

| 功能 | 需求 | 优先级 |
|------|------|--------|
| 2D形象展示 | 宠物虚拟形象动态展示 | P0 |
| 宠物状态 | 活跃/睡觉/进食（硬件数据驱动） | P0 |
| 重要事项提醒 | 绝育、疫苗、驱虫、运动 | P0 |
| 宠物铃 | 长按发语音交互 | P0 |

#### 底部导航
- 左：宠物状态展示页面
- 中：AI Agent对话页（"小能"）
- 右：我的

### 3.6 AI Agent（小能）

| 功能 | 需求 | 优先级 |
|------|------|--------|
| 自然语言对话 | 聊天式交互 | P0 |
| 信息录入 | 通过对话记录宠物信息（饭量、排便、粮品牌等） | P0 |
| 数据Mock | 自动将聊天数据同步到前端展示 | P0 |
| 主动询问 | 对未录入信息主动询问用户 | P1 |

| 功能 | 优先级 |
|------|--------|
| 硬件设置 | P1 |
| 购买登记 | P2 |
| 修改信息 | P1 |
| 添加宠物 | P0 |
| 问题与反馈 | P1 |
| 参与产品调研 | P2 |
| 个人隐私条款和免责声明 | P0 |
| 退出登录 | P0 |

### 3.8 提醒系统

| 提醒类型 | 触发方式 | 优先级 |
|----------|----------|--------|
| 绝育提醒 | 用户录入后定时推送 | P1 |
| 疫苗提醒 | 用户录入后定时推送 | P1 |
| 驱虫提醒 | 用户录入后定时推送 | P1 |
| 运动提醒 | AI Agent主动询问 | P2 |
| 状态异常 | 硬件数据检测后推送 | P1 |

## 4. 数据需求

### 4.1 硬件数据（真实）
- 宠物状态（活跃/睡觉/进食）
- 睡眠时长
- 活动量

**数据来源：** 蓝牙/Wi-Fi设备同步

### 4.2 用户录入数据
- 宠物基础信息
- 疫苗/驱虫/绝育记录
- 日常习惯（饭量、排便、粮品牌）

**录入方式：** AI Agent对话式录入
